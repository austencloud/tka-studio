Appearing\n1. Ensure element has `overflow: auto` or `overflow-y: auto`\n2. Check that content actually overflows the container\n3. Verify the glass scrollbar CSS is imported in `app.css`\n\n### Styling Conflicts\n1. Remove existing `::-webkit-scrollbar` rules\n2. Use `:global()` when targeting from Svelte components\n3. Check CSS specificity - glass scrollbar styles should have higher specificity\n\n### Performance Issues\n1. Use `scroll-behavior: auto` on mobile (automatically handled)\n2. Reduce `backdrop-filter` blur amount if needed\n3. Use `minimal` variant for better performance\n\n## üì¶ Files\n\n- **`/lib/styles/glass-scrollbars.css`** - Core CSS styles\n- **`/lib/components/ui/GlassScrollContainer.svelte`** - React component wrapper\n- **`/lib/utils/glassScrollbars.ts`** - Utility functions\n- **`/app.css`** - Import location\n\n## üé® Examples in Action\n\n### Codex Component\n```svelte\n<GlassScrollContainer variant=\"primary\" height=\"100%\">\n\t<CodexPictographGrid />\n</GlassScrollContainer>\n```\n\n### Option Picker\n```html\n<div class=\"option-picker-scroll glass-scrollbar-secondary\">\n\t<!-- options -->\n</div>\n```\n\n### Browse Tab (Recommended)\n```svelte\n<div class=\"browse-content\" data-glass-scrollbar=\"primary\">\n\t<!-- browse content -->\n</div>\n```\n\nThat's it!
