NAVIGATION ARCHITECTURE
========================================

Current Pattern: Top Bar + Adaptive Bottom/Side Navigation + Modal Menu

Flow:
  MainInterface.svelte
    |
    +-- TopBar (always visible)
    |   +-- Profile button
    |   +-- Gamification button
    |   +-- Module-specific content slot
    |
    +-- UnifiedNavigationMenu (modal, always rendered)
    |   +-- ModuleList
    |   +-- InstallPromptButton
    |
    +-- PrimaryNavigation (conditional, if module has nav)
        +-- Module switcher (opens menu)
        +-- Section buttons (module-specific)
        +-- Settings button

KEY FINDINGS
========================================

1. CONSOLIDATED DESIGN
   - Single top bar approach
   - Single primary navigation (adapts to device)
   - Single module switcher menu
   - No complex dropdown patterns

2. ADAPTIVE LAYOUT
   - PrimaryNavigation switches between:
     * Bottom bar: 64px height, horizontal layout (portrait)
     * Side bar: 72px width, vertical layout (landscape)
   - Responsive labels: full (600px+) / compact (450-599px) / icons only (<450px)

   - 12 unused navigation components

4. MODAL-BASED SHEETS
